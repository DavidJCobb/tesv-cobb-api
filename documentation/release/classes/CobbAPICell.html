CobbAPICell

These functions allow you to work with cells. In particular, you can modify the lighting and fog properties of any interior cell that doesn't use sky lighting, in a manner that persists in the SKSE co-save.

An overview of interior cell lighting

Interior cells have a wide array of lighting properties, allowing for precise control of fog and color. However, an interior cell can also use a lighting template — a separate form that offers the same range of lighting properties. A cell can borrow some or all of its values from its lighting template.

Internally, cells have a set of lighting template usage flags — a bitmask indicating which properties are borrowed from the template. Cells always define the full set of lighting data on their end, but when the game goes to look up a lighting property for the cell, it will first check the template usage flags; if the flag for the property in question is set, then the game will instead grab the property value from the lighting template. If the cell somehow has a template usage flag set without actually having a lighting template to pull from, then the game will still use the cell-side property. Properties are looked up every frame, so any changes you make will be instantaneous (not counting the time it takes for the Papyrus API to actually execute).

If you wish to reset your changes to a cell's lighting properties, please use the "reset" functions. If you get a cell's original values and then set it to those values manually, you will be baking those values into the save; if a mod changes that cell's values, the values you set will take precedence. The "reset" functions avoid this.

Lighting template usage flag list

The lighting template usage flags that cells can have are as follows:

Flag Properties
1 Ambient color
2 Directional color
4 Fog color (both near and far)
8 Fog near distance
16 Fog far distance
32 Directional rotation
64 Directional fade
128 Fog clip distance
256 Fog power
512 Fog max
1024 Light fade distances
